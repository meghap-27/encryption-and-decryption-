Encryption Algorithms Explained

Caesar Cipher

Complexity: O(n) | Type: Substitution Cipher

The Caesar cipher is one of the simplest and most widely known encryption techniques. It's a substitution cipher where each letter in the plaintext is shifted a certain number of places down the alphabet.

char caesarEncrypt(char ch, int shift) { if (isalpha(ch)) { char base = isupper(ch) ? 'A' : 'a'; return (ch - base + shift) % 26 + base; } return ch; }
  • Historical significance in cryptography
  • Easy to understand and implement
  • Vulnerable to brute force (only 25 possible keys)
  • Foundation for understanding substitution ciphers

Vigen√®re Cipher

Complexity: O(n) | Type: Polyalphabetic Cipher

The Vigen√®re cipher uses a keyword to shift letters by different amounts, making it more secure than Caesar. Each letter of the keyword determines the shift for the corresponding plaintext letter.

string vigenereEncrypt(string text, string key) { string result = ""; for (int i = 0; i < text.length(); i++) { int shift = key[i % key.length()] - 'A'; result += caesarEncrypt(text[i], shift); } return result; }
  • More secure than simple substitution ciphers
  • Uses polyalphabetic substitution
  • Resistant to frequency analysis
  • Keyword length affects security strength

RSA Encryption

Complexity: O(log n) | Type: Asymmetric Encryption

RSA is a public-key cryptosystem widely used for secure data transmission. It uses two keys: a public key for encryption and a private key for decryption, based on the mathematical properties of prime numbers.

// Key Generation n = p * q; phi = (p - 1) * (q - 1); // Choose e: gcd(e, phi) = 1 // Calculate d: (d * e) mod phi = 1 // Encryption: c = m^e mod n // Decryption: m = c^d mod n
  • Foundation of modern secure communication
  • Uses public and private key pairs
  • Based on prime factorization difficulty
  • Essential for SSL/TLS, digital signatures
